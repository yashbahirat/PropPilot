# 02-07: Seed Script

## Goal
Create a Prisma seed script to populate the database with realistic example prop firms.

## Context
- Requirement `DB-03` requires at least 8 realistic example prop firms (e.g., FTMO-style, Apex-style, etc.).

## Tasks

<task>
<read_first>
- package.json
- prisma/seed.ts
</read_first>
<action>
Create or update `prisma/seed.ts`.
- Write an async function that clears existing data (optional, or upserts to be safe).
- Define an array of 8 realistic firm objects matching the `Firm` schema fields (name, slug, challengeFee, profitTarget, maxDrawdown, etc.).
- Loop over the array and use `db.firm.upsert` to insert the firms.
- Include a few example `FirmOffer` records linked to these firms.
</action>
<acceptance_criteria>
- `seed.ts` contains `db.firm.upsert`
- `seed.ts` contains 8 firm objects
</acceptance_criteria>
</task>

<task>
<read_first>
- package.json
</read_first>
<action>
Update `package.json` to include the Prisma seed command:
`"prisma": { "seed": "ts-node --compiler-options {\\"module\\":\\"CommonJS\\"} prisma/seed.ts" }`
Ensure `ts-node` is installed as a devDependency.
</action>
<acceptance_criteria>
- `package.json` contains `"prisma": { "seed": ... }`
</acceptance_criteria>
</task>

## Verification
1. Running `npx prisma db seed` successfully populates the database without errors.
2. The newly seeded firms are visible in the admin dashboard at `/admin/firms`.
