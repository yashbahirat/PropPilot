# PropPilot

> The trusted comparison platform for proprietary trading firms.

Compare prop firms, discover verified discount codes, and earn rewards for verified purchases.

## Tech Stack

- **Framework:** Next.js 15 (App Router)
- **Language:** TypeScript (strict)
- **Styling:** Tailwind CSS v4 + shadcn/ui
- **Auth:** Clerk

## Getting Started

### 1. Clone and install

```bash
git clone <repo-url>
cd PropPilot
npm install
```

### 2. Configure environment variables

Copy `.env.example` to `.env.local` and fill in all values:

```bash
cp .env.example .env.local
```

Required environment variables:

| Variable | Source | Description |
|----------|--------|-------------|
| `DATABASE_URL` | Neon Dashboard | Pooled connection string |
| `DIRECT_URL` | Neon Dashboard | Direct connection (migrations) |
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk Dashboard | Public Clerk key |
| `CLERK_SECRET_KEY` | Clerk Dashboard | Secret Clerk key |
| `CLERK_WEBHOOK_SECRET` | Clerk Webhooks | Svix signing secret |
| `UPLOADTHING_SECRET` | UploadThing Dashboard | File upload secret |
| `UPLOADTHING_APP_ID` | UploadThing Dashboard | App ID |
| `NEXT_PUBLIC_APP_URL` | — | `http://localhost:3000` (dev) |

### 3. Push Prisma schema to database

```bash
npx prisma db push
npx prisma generate
```

### 4. Configure Clerk webhook (for user sync)

1. Go to [Clerk Dashboard](https://clerk.com) → Webhooks → Add Endpoint
2. URL: `https://your-domain.com/api/webhooks/clerk`
3. Events: `user.created`, `user.updated`
4. Copy Signing Secret to `CLERK_WEBHOOK_SECRET` in `.env.local`

### 5. Run development server

```bash
npm run dev
```

Visit [http://localhost:3000](http://localhost:3000)

## Deploying to Vercel

1. Push to GitHub
2. Connect repo to Vercel
3. Add all environment variables from `.env.example` in Vercel Project Settings → Environment Variables
4. Update Clerk webhook URL to your Vercel deployment URL
5. Deploy

## Architecture

- **Route protection:** Clerk middleware (`middleware.ts`) protects `/dashboard` and requires admin role for `/admin`
- **User sync:** Clerk webhook at `/api/webhooks/clerk` creates User + UserRewardPoints in DB on sign-up
- **Affiliate tracking:** `/go/[slug]` redirect route (Phase 2) logs clicks asynchronously
- **Design system:** PropPilot custom palette (teal/cyan/purple) on dark base — see `tailwind.config.ts`

## Security Threat Model

| Threat | Severity | Mitigation |
|--------|----------|-----------|
| Environment variables exposed in Vercel build logs | High | Vercel masks env vars in logs by default; never log env vars in code |
| `DATABASE_URL` (pooled) used for migrations on Vercel | High | `DIRECT_URL` is used by Prisma for migrations; `DATABASE_URL` for app queries |
| Production Clerk secret key exposed in README | Critical | README uses placeholders — actual keys only in `.env.local` and Vercel dashboard |
| Unauthorized API access to webhook endpoint | High | Svix signature verification (Plan 07) prevents unauthorized event processing |
| Next.js build errors on Vercel due to missing env vars | Medium | `vercel.json` sets `SKIP_ENV_VALIDATION=1` for build; runtime validation handles missing vars gracefully |
</threat_model>
