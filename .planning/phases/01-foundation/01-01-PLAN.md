# Plan 01: Next.js 15 Project Initialization

## Objective
Bootstrap the PropPilot Next.js 15 project with TypeScript strict mode, App Router, Tailwind CSS, and ESLint. This is the foundational scaffold every other plan builds on.

## Tasks

<task id="1">
<action>
Run create-next-app with all required flags to scaffold the project in the current directory:

```bash
npx create-next-app@latest ./ \
  --typescript \
  --tailwind \
  --eslint \
  --app \
  --src-dir \
  --import-alias "@/*" \
  --no-git
```

Accept all defaults. This creates: `src/app/`, `public/`, `next.config.ts`, `tsconfig.json`, `package.json`, `.eslintrc.json`, `postcss.config.mjs`.
</action>
<read_first>
- Check if package.json already exists before running (would indicate project already initialized)
</read_first>
<acceptance_criteria>
- `package.json` contains `"next": "^15"` in dependencies
- `tsconfig.json` contains `"strict": true` in compilerOptions
- `src/app/` directory exists with `layout.tsx` and `page.tsx`
- `tailwind.config.ts` exists (created by create-next-app)
- Running `npx tsc --noEmit` exits 0 (no type errors)
</acceptance_criteria>
</task>

<task id="2">
<action>
Update `next.config.ts` to configure Next.js 15 with the following settings:

```typescript
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    // Required for Clerk + Next.js 15 compatibility
  },
  images: {
    remotePatterns: [
      { protocol: 'https', hostname: '**.clerk.com' },
      { protocol: 'https', hostname: '**.uploadthing.com' },
      { protocol: 'https', hostname: 'img.clerk.com' },
      { protocol: 'https', hostname: 'uploadthing.com' },
    ],
  },
}

export default nextConfig
```
</action>
<read_first>
- `next.config.ts` (current default content from create-next-app)
</read_first>
<acceptance_criteria>
- `next.config.ts` contains `remotePatterns` with `clerk.com` and `uploadthing.com` entries
- `next.config.ts` does NOT contain `experimental.serverActions: true` (enabled by default in Next.js 15)
- File is valid TypeScript — `npx tsc --noEmit` exits 0
</acceptance_criteria>
</task>

<task id="3">
<action>
Update `tsconfig.json` to enforce strict TypeScript settings required for production-quality PropPilot code:

In `compilerOptions`, ensure these are set:
```json
{
  "strict": true,
  "noUncheckedIndexedAccess": true,
  "noImplicitReturns": true,
  "exactOptionalPropertyTypes": false,
  "paths": {
    "@/*": ["./src/*"]
  }
}
```

Also add to `include`: `["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts"]`
</action>
<read_first>
- `tsconfig.json` (current content from create-next-app)
</read_first>
<acceptance_criteria>
- `tsconfig.json` contains `"strict": true`
- `tsconfig.json` contains `"noUncheckedIndexedAccess": true`
- `tsconfig.json` contains `"noImplicitReturns": true`
- `tsconfig.json` contains `"@/*": ["./src/*"]` in paths
</acceptance_criteria>
</task>
